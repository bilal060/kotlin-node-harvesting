📱 MOBILE DATA ACCESS CAPABILITIES
==========================================
Timestamp: $(date)

This document outlines all the data types and structures that can be accessed
from an Android mobile device through our app.

📧 EMAIL ACCOUNTS
------------------------------------------
Data Structure:
{
  "name": "user@example.com",           // Email address
  "type": "com.google",                 // Account type (com.google, com.microsoft, etc.)
  "description": "",                    // Account description (often empty due to permissions)
  "account_info": {
    "last_authenticated_time": "",      // Last login time (often empty due to permissions)
    "password": ""                      // Always empty for security reasons
  }
}

Accessible Fields:
✅ Email address (name)
✅ Account type (type)
❌ Description (requires special permissions)
❌ Last authenticated time (requires special permissions)
❌ Password (never accessible for security)

==========================================
📱 SUMMARY OF ACCESSIBLE DATA
==========================================

✅ FULLY ACCESSIBLE:
- Contact information (names, phones, emails, addresses)
- Call logs (numbers, durations, timestamps)
- Notifications (app, title, content, timestamps)
- Email account addresses and types
- Installed apps (names, versions, sizes)
- File information (paths, sizes, types)
- Calendar events (titles, times, locations)
- Browser history (URLs, titles, visit times)
- Location data (GPS coordinates, accuracy)
- Network information (WiFi, mobile, Bluetooth)
- System information (device, battery, storage)
- Hardware information (RAM, CPU, sensors)

⚠️ PARTIALLY ACCESSIBLE:
- Email account descriptions (requires special permissions)
- Email account authentication times (requires special permissions)
- Some system settings (requires special permissions)
- Some hardware details (requires special permissions)

❌ NOT ACCESSIBLE:
- Account passwords (never accessible for security)
- Encrypted data
- System-protected information
- Other apps' private data
